## 1. 这个项目适合学什么

FilterPy 适合用来学习机器人中的状态估计问题，尤其是：

- Kalman Filter
- Extended Kalman Filter
- Unscented Kalman Filter
- Particle Filter
- 传感器噪声建模
- 轨迹估计与多传感融合

对应机器人方向：

- SLAM / VIO
- 移动机器人定位
- 自动驾驶状态估计
- 多传感器融合
- 闭环控制中的状态反馈

对你现在最有价值的定位：

> 用 FilterPy 做一个“移动机器人多传感器状态估计 demo”，把你的光学装调、视觉测量、闭环控制背景自然过渡到机器人感知与控制方向。

## 2. 新手应该先怎么做

不要一开始就看完整源码，也不要直接冲 EKF/VIO。建议按下面顺序来：

### Step 1：先理解状态估计在机器人里解决什么问题

先搞清楚三个概念：

- 状态：机器人当前的位置、速度、姿态等，例如 `[x, y, vx, vy]`
- 预测：根据运动模型推测下一时刻状态
- 更新：用传感器观测修正预测结果

机器人例子：

```text
机器人根据轮速计预测自己移动了多少
相机或 GPS 给出一个带噪声的位置观测
Kalman Filter 把两者融合，得到更稳定的位置估计
```

这一阶段目标：

- 知道 `predict` 和 `update` 分别在做什么。
- 知道过程噪声 `Q`、观测噪声 `R`、协方差 `P` 的直观意义。
- 能解释为什么融合后的轨迹比单一传感器更稳定。

### Step 2：先跑一维 Kalman Filter

最简单任务：

> 估计一个沿直线运动的小车位置。

状态可以定义为：

```text
x = [position, velocity]
```

观测只有：

```text
z = measured_position
```

你需要做：

- 生成真实位置曲线。
- 添加高斯噪声作为传感器观测。
- 用 Kalman Filter 估计位置和速度。
- 画出真实值、观测值、估计值。

完成标准：

- 能跑出一张图。
- 图上有 ground truth、noisy measurement、filtered estimate。
- 能说清楚为什么估计结果更平滑。

### Step 3：扩展到二维移动机器人

二维机器人状态：

```text
x = [px, py, vx, vy]
```

运动模型：

```text
px_new = px + vx * dt
py_new = py + vy * dt
```

观测模型：

```text
z = [measured_px, measured_py]
```

你需要做：

- 生成一条 2D 轨迹，例如圆形、S 形或折线路径。
- 模拟视觉/GPS 位置观测噪声。
- 用 Kalman Filter 估计轨迹。
- 输出轨迹图和 RMSE。

完成标准：

- 有 2D 轨迹图。
- 有误差曲线。
- 有 RMSE 指标。

### Step 4：加入机器人传感器语境

把普通数学例子改成机器人语境：

```text
真实轨迹：ground truth
轮速计：odometry prediction
视觉定位：camera measurement
滤波结果：fused robot pose
```

你需要做：

- 用轮速计作为预测输入。
- 用视觉位置作为观测更新。
- 模拟视觉观测偶尔丢失。
- 比较只有 odometry、只有 camera、融合结果三种轨迹。

完成标准：

- 输出 3 条轨迹对比。
- 输出视觉丢失情况下的估计结果。
- 写一段分析：为什么融合比单传感器更可靠。

### Phase 2：EKF 与非线性机器人模型

目标：

- 理解为什么机器人常用 EKF。
- 用 EKF 估计带朝向的机器人位姿。

机器人状态：

```text
x = [px, py, theta]
```

控制输入：

```text
u = [v, omega]
```

运动模型：

```text
px_new = px + v * cos(theta) * dt
py_new = py + v * sin(theta) * dt
theta_new = theta + omega * dt
```
